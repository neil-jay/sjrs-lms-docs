This directory contains all SQL scripts for the SJRS Library Management System.

## Directory Structure

### [Core Files](./)
Main database schema and documentation:
- **[d1-schema.sql](./d1-schema.sql)** - Complete D1 database schema (SQLite compatible) with all tables, indexes, triggers, and initial data
- **[index.md](./index)** - This documentation file

### [Migrations](./migrations/)
Applied database migrations that have been integrated into the main schema:
- See [migrations/index.md](./migrations/index) for detailed status

## Usage Guide

### **Initial Setup (First Time)**
1. **Run `d1-schema.sql`** - This single file creates the complete database with:
   - All tables (30+ tables)
   - All indexes for performance
   - All triggers for data integrity
   - Initial permission system data
   - Default roles and basic configuration

### **No Additional Setup Required**
- All permission tables are included in the main schema
- All initial data is pre-populated
- The system is ready to use immediately after running the main schema

## **Schema Overview**

### **Core Tables (30+ tables)**
- **User Management**: `roles`, `library_users`, `students`, `user_sessions`, `user_mfa_settings`
- **Permission System**: `permission_resources`, `permission_actions`, `role_permissions`, `permission_audit_log`
- **Book Management**: `books`, `book_copies`, `authors`, `publications`, `book_reviews`
- **Reference System**: `reference_books`, `resource_types`, `reference_categories`
- **Academic**: `sections`, `journals`, `journal_articles`, `journal_authors`, `journal_tags`
- **Circulation**: `loans`, `orders`, `penalties`, `borrow_limits`
- **Finance**: `payments`, `receipts`
- **System**: `notifications`, `action_logs`, `system_logs`, `rate_limits`

### **Features**
- **Complete permission system** with role-based access control
- **Full-text search** using FTS5
- **Optimized indexes** for performance
- **Data integrity triggers** for timestamps
- **Views** for common queries
- **Initial data** for immediate use

## **Benefits of Single Schema File**

1. ** Single Source of Truth**: One file contains everything needed
2. ** No Dependencies**: No need to run multiple setup scripts
3. ** Self-Contained**: Complete system in one file
4. ** Easy Maintenance**: All changes in one place
5. ** Quick Setup**: New environments ready in minutes

---

**Note**: All scripts are optimized for D1 (SQLite) and are not compatible with PostgreSQL/Supabase.
