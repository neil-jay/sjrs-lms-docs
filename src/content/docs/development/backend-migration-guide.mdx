## Overview

The backend now lives under `functions/` as direct function APIs (Cloudflare Workers native). This guide summarizes the completed migration to the **standardized API response system** and the move away from framework-based routing. All existing code continues to work thanks to backward compatibility features.

## **Migration Status: COMPLETE**

Good news! **No immediate action required.** The migration has been completed and all existing code continues to work.

### **What Was Changed:**
- **Response Builder**: New centralized response creation system
- **Standardized Format**: All responses follow consistent structure
- **Backward Compatibility**: All existing handlers continue to work
- **Type Safety**: Full TypeScript support throughout
- **Error Handling**: Improved error responses with codes
 -  **Direct Function APIs**: Replaced framework routing with `functions/api/*` modules
 -  **Unified Entry**: Worker entry at `functions/index.ts` with module routers

### **What Wasn't Broken:**
- **Existing Handlers**: All continue to work without changes
- **Frontend Integration**: All React hooks work seamlessly
- **API Contracts**: No breaking changes to API endpoints
- **Authentication**: All auth flows remain the same

## **For New Development**

When creating **new API endpoints**, use the standardized response builder:

### **Where to Add Endpoints**
New endpoints belong under `functions/api/<feature>/` following the standardized module pattern. See `docs/development/functions-folder-architecture.md` for structure and examples.

### **Quick Start**
```typescript
// Import the standardized response functions
import { 
  createSuccessResponse, 
  createErrorResponse,
  createPaginatedResponse,
  createCreatedResponse 
} from '../../../utilities/response-builder';

// Use in your handlers
export async function handleNewEndpoint(
  request: Request, 
  env: Environment,
  user: any,
  sanitizedData?: any
): Promise<Response> {
  const origin = request.headers.get('Origin');
  
  try {
    // Use pre-validated data from security middleware
    const data = sanitizedData ?? await request.json();
    
    // ... your logic ...
    
    return createSuccessResponse(
      data,                              // Your response data
      'Operation completed successfully', // Message
      200,                              // Status code (optional)
      { version: '1.0.0' },            // Metadata (optional)
      origin                            // Origin for CORS
    );
  } catch (error) {
    return createErrorResponse(
      'Operation failed',               // User message
      500,                             // Status code
      'INTERNAL_ERROR',               // Error code
      error.message,                  // Details
      undefined,                      // Validation fields
      origin                          // Origin for CORS
    );
  }
}
```

## **Optional Improvements**

While not required, you can **optionally** update existing handlers to use more specific response functions:

### **Before (Still Works)**

## **Response Format Reference**

### **Old Format (Still Supported)**
```typescript
// Basic success response
{
  data: any
}

// Basic error response  
{
  error: string
}
```

### **New Standardized Format**
```typescript
// Success response
{
  "success": true,
  "data": any,                         // Your data
  "message": string,                   // Human-readable message
  "status": number,                    // HTTP status code
  "timestamp": "2024-01-15T10:30:00Z", // ISO timestamp
  "requestId": "req_1705312200_abc123", // Unique request ID
  "meta": {                            // Optional metadata
    "processingTime": 45,
    "version": "1.0.0"
  }
}

// Error response
{
  "success": false,
  "message": string,                   // User-friendly message
  "error": {
    "code": "VALIDATION_ERROR",        // Error code
    "details": string,                 // Detailed error info
    "fields": {                        // Field validation errors
      "email": ["Email is required"]
    }
  },
  "status": number,                    // HTTP status code
  "timestamp": "2024-01-15T10:30:00Z", // ISO timestamp
  "requestId": "req_1705312200_err456"  // Unique request ID
}
```

## **Error Codes Reference**

Use these standardized error codes for consistent error handling:

```typescript
ERROR_CODES = {
  VALIDATION_ERROR: 'VALIDATION_ERROR',    // Field validation failures
  NOT_FOUND: 'NOT_FOUND',                  // Resource not found (404)
  UNAUTHORIZED: 'UNAUTHORIZED',            // Authentication required (401)
  FORBIDDEN: 'FORBIDDEN',                  // Access denied (403)
  CONFLICT: 'CONFLICT',                    // Resource conflict (409)
  INTERNAL_ERROR: 'INTERNAL_ERROR',        // Server errors (500)
  RATE_LIMITED: 'RATE_LIMITED',            // Too many requests (429)
  BAD_REQUEST: 'BAD_REQUEST',              // Invalid request (400)

}
```

## **Best Practices**

### **1. Always Include Origin for CORS**
```typescript
request.headers.get('Origin')
```

### **2. Use Appropriate Error Codes**
```typescript
// For validation errors
createErrorResponse(message, 422, 'VALIDATION_ERROR', details, fields, origin);

// For not found errors
createErrorResponse(message, 404, 'NOT_FOUND', details, undefined, origin);
```

### **3. Include Meaningful Metadata**
```typescript
createSuccessResponse(data, message, 200, {
  processingTime: Date.now() - startTime,
  version: '1.0.0',
  filters: appliedFilters
}, origin);
```

### **4. Use Specialized Functions When Appropriate**
```typescript
// For creation operations
return createCreatedResponse(newResource, 'Resource', origin);

// For paginated lists
return createPaginatedResponse(items, paginationInfo, message, meta, origin);
```

## **Example Migration**

### **Old Handler (Still Works)**
```typescript
export async function handleOldEndpoint(request: Request, env: any): Promise<Response> {
  try {
    const result = await someOperation();
    const response = new Response(JSON.stringify({ data: result }), {
      headers: { 'Content-Type': 'application/json' }
    });
    return addCORSHeaders(response, request.headers.get('Origin'));
  } catch (error) {
    const errorResponse = new Response(JSON.stringify({ error: 'Failed' }), {
      status: 500,
      headers: { 'Content-Type': 'application/json' }
    });
    return addCORSHeaders(errorResponse, request.headers.get('Origin'));
  }
}
```

### **New Handler (Recommended)**
```typescript
import { createSuccessResponse, createErrorResponse } from '../../../utilities/response-builder';

      request.headers.get('Origin')
    );
  } catch (error) {
    return createErrorResponse(
      'Operation failed',
      500,
      'INTERNAL_ERROR',
      error.message,
      undefined,
      request.headers.get('Origin')
    );
  }
}
```
