## Known Patterns & Conventions

1. **Query Key Namespacing**: `['entity', 'subtype', params]`
2. **Preset Assignment Rules**:
   - Dashboard/stats data → REAL_TIME (always fresh)
   - User lists/permissions → FREQUENT (30s cache)
   - Reference data (roles, categories) → STATIC (5min cache)
   - Immutable data → IMMUTABLE (only manual invalidation)
3. **Mutation Success Pattern**: Invalidate + refetch active queries
4. **Error Handling**: All queries integrated with unified error handler
5. **Pagination Pattern**: `keepPreviousData` + server-side setup

## Usage Examples

### Basic Query Hook

```typescript
import { useStandardQuery } from '../../utilities/query';

export const useRolesQuery = () => {
  return useStandardQuery({
    queryKey: ['permissions', 'roles'],
    queryFn: async () => {
      const roles = await getAllRoles();
      return Array.isArray(roles) ? roles : [];
    },
    preset: 'STATIC', // 5min cache for rarely-changing data
  });
};
```

### Query with Pagination

```typescript
import { useStandardQuery } from '../../utilities/query';
import { keepPreviousData } from '@tanstack/react-query';

export const useStudentsQuery = (page: number, filters: Filter[]) => {
  return useStandardQuery({
    queryKey: ['students', page, filters],
    queryFn: async () => {
      return await fetchStudents({ page, filters });
    },
    preset: 'FREQUENT', // 30s cache for frequently-changing data
    placeholderData: keepPreviousData, // Prevent UI flicker
  });
};
```

### Mutation with Cache Invalidation

```typescript
import { useStandardMutation } from '../../utilities/query';

export const useCreateStudent = () => {
  return useStandardMutation({
    mutationFn: async (data: StudentData) => {
      return await createStudent(data);
    },
    invalidateKeys: [['students']], // Invalidate student queries
    refetchKeys: [['students']], // Refetch active student queries
    onSuccess: (data) => {
      message.success('Student created successfully');
    },
  });
};
```
