## Major Architectural Changes

### 1. Instant-Close Modal Pattern (v6.0.0)

#### Breaking Change
All CRUD modals now close immediately before API response for improved UX.

#### Implementation
```typescript
// Before: Wait for API response
const handleSubmit = async (values) => {
  await mutation.mutateAsync(values);
  closeModal();
};

// After: Instant close pattern
const handleSubmit = useInstantCloseMutation({
  mutation: updateMutation,
  record: editingRecord,
  config: {
    onClose: editModal.close,
    setRefreshTime: onRefresh,
    reopenModal: (record, formData) => {
      editModal.form.setFieldsValue(formData);
      editModal.open(record);
    },
  },
});
```

#### Benefits
- **0ms Perceived Latency**: Modal closes instantly
- **Background Processing**: API calls execute non-blocking
- **Error Recovery**: Auto-reopens modal on failure
- **Consistent UX**: Uniform behavior across all CRUD operations

### 2. Centralized Query Configuration (v5.0.0)

#### Breaking Change
All query hooks now use centralized `useStandardQuery` wrapper.

#### Migration Summary
- **57 files modified** (38 files use `useStandardQuery`, 91 `useStandardMutation` instances)
- **0 raw `useQuery/useMutation`** in production code
- **Net reduction**: 72 lines of code removed
- **Centralized presets**: REAL_TIME, FREQUENT, STATIC, IMMUTABLE

#### Implementation
```typescript
// Before: Scattered configuration
const { data } = useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
  staleTime: 30000,
  gcTime: 300000,
});

// After: Centralized configuration
const { data } = useStandardQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
  preset: 'FREQUENT',
});
```

#### Presets
```typescript
const QUERY_PRESETS = {
  REAL_TIME: { staleTime: 0, refetchOnMount: true },      // Dashboards, live data
  FREQUENT: { staleTime: 30000, refetchOnMount: true },  // User lists, catalogs
  STATIC: { staleTime: 300000, refetchOnMount: false },  // Reference data, settings
  IMMUTABLE: { staleTime: Infinity, refetchOnMount: false }, // Historical data
};
```

## Performance Improvements

### 1. Optimistic Updates (v6.1.0)

#### Implementation
All CRUD operations now implement optimistic updates:

```typescript
// Create operation
onMutate: async (data) => {
  await queryClient.cancelQueries({ queryKey: ['resource'] });
  const previous = queryClient.getQueryData(['resource']);
  
  // Optimistic insert
  const tempId = -Date.now();
  const optimisticItem = { ...data, id: tempId };
  queryClient.setQueryData(['resource'], (prev) => ({
    ...prev,
    data: [optimisticItem, ...prev.data],
  }));
  
  return { previous, tempId };
},

onError: (error, data, context) => {
  if (context?.previous) {
    queryClient.setQueryData(['resource'], context.previous);
  }
},

onSuccess: (result, data, context) => {
  // Replace temp item with server response
  queryClient.setQueryData(['resource'], (prev) => ({
    ...prev,
    data: prev.data.map(item => 
      item.id === context?.tempId ? result.data : item
    ),
  }));
},
```

## Migration Guides

### 1. Modal Pattern Migration

#### Steps
1. **Install**: Import `useCrudModalState` and `useInstantCloseMutation`
2. **Replace**: Remove manual modal state management
3. **Implement**: Add optimistic updates to hooks
4. **Test**: Verify instant-close behavior

#### Example
```typescript
// Old approach
const [modalVisible, setModalVisible] = useState(false);
const [form] = Form.useForm();

const handleSubmit = async (values) => {
  await mutation.mutateAsync(values);
  setModalVisible(false);
  refetch();
};

// New approach
const { createModal, editModal, onRefresh } = useCrudModalState<Entity>({
  setRefreshTime: setLastRefreshTime,
  refetch,
});

const handleSubmit = useInstantCloseMutation({
  mutation: createMutation,
  record: { id: 0 } as Entity,
  config: {
    onClose: createModal.close,
    setRefreshTime: onRefresh,
  },
});
```

### 2. Query Configuration Migration

#### Steps
1. **Identify**: Find all raw `useQuery`/`useMutation` usage
2. **Replace**: Use `useStandardQuery`/`useStandardMutation`
3. **Configure**: Select appropriate preset
4. **Remove**: Delete hardcoded staleTime/gcTime values

#### Example
```typescript
// Before
const { data } = useQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
  staleTime: 30000,
  gcTime: 300000,
  refetchOnWindowFocus: true,
});

// After
const { data } = useStandardQuery({
  queryKey: ['users'],
  queryFn: fetchUsers,
  preset: 'FREQUENT',
});
```

## Troubleshooting

### Common Issues

#### 1. Modal Not Closing
- **Cause**: Not using `useInstantCloseMutation`
- **Solution**: Implement instant-close pattern
- **Check**: Verify `onClose` callback is properly set

#### 2. Query Not Updating
- **Cause**: Missing optimistic updates
- **Solution**: Add `onMutate`/`onError`/`onSuccess` handlers
- **Check**: Verify query key consistency

#### 3. Permission Errors
- **Cause**: Missing permission checks
- **Solution**: Add proper permission gates
- **Check**: Verify role assignments
