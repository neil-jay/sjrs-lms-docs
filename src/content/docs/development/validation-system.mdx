## Overview

The SJRS LMS features a **modern, modular validation system** designed for performance, security, and maintainability. The system provides comprehensive input validation and sanitization across the frontend application.

## Architecture

### **Clean Modular Design**
```
src/utilities/validation/
├── core/
│   ├── engine.ts           # Main validation orchestration
│   ├── cache-manager.ts    # Performance optimization
│   └── config.ts           # Configuration management
├── validators/
│   ├── basic-validators.ts    # String, number, boolean
│   ├── format-validators.ts   # Email, phone, URL, ISBN
│   ├── password-validators.ts # Password complexity
│   └── security-validators.ts # Security threat detection
├── utils/
│   └── error-creators.ts   # Result creation utilities
├── types.ts                # TypeScript definitions
└── index.ts                # Single entry point
```

## Quick Start

### **Basic Usage**
```typescript
import { validate, validateForm } from '@/utilities/validation';

// Single field validation
const emailResult = validate(email, { 
  type: 'email', 
  required: true 
});

// Form validation
const schema = {
  email: { type: 'email', required: true },
  password: { type: 'password', required: true },
  name: { type: 'name', required: true }
};

const formResult = validateForm(formData, schema);
```

### **React Hook Integration**
```typescript
// Primary usage via useInputValidation hook
import { useInputValidation } from '@/hooks/useInputValidation';

const { validateField, validateForm, fieldErrors, isValid } = useInputValidation({
  schema: {
    email: { type: 'email', required: true },
    password: { type: 'password', required: true }
  }
});
```

## Security Features

### **Comprehensive Threat Detection**
- **XSS Prevention**: Detects and blocks cross-site scripting attempts
- **SQL Injection Protection**: Identifies malicious SQL patterns
- **Command Injection Protection**: Blocks system command execution attempts
- **Path Traversal Protection**: Prevents directory traversal attacks
- **LDAP Injection Protection**: Detects LDAP injection patterns
- **NoSQL Injection Protection**: Blocks NoSQL injection attempts

### **Security Validation Example**
```typescript
import { SecurityValidators } from '@/utilities/validation';

const securityResult = SecurityValidators.validateSecurity(userInput);

  console.warn('Security threat detected:', securityResult.errors);
}
```

## Validation Rules

### **Basic Rules**
```typescript
interface ValidationRule {
  required?: boolean;        // Field is required
  minLength?: number;        // Minimum string length
  maxLength?: number;        // Maximum string length
  min?: number;             // Minimum numeric value
  max?: number;             // Maximum numeric value
  pattern?: RegExp;         // Custom regex pattern
  trimWhitespace?: boolean; // Auto-trim whitespace
  custom?: (value: any) => boolean | string; // Custom validation
}
```

### **Type-Specific Validation**
```typescript
// Email validation
{ type: 'email', required: true }

// Password with complexity requirements
{ type: 'password', required: true }

// Phone number validation
{ type: 'phone' }

// URL validation
{ type: 'url' }

// ISBN validation (10 or 13 digits)
{ type: 'isbn' }

// Year validation (1900 to current+10)
{ type: 'year' }

// Name validation (letters, spaces, hyphens)
{ type: 'name', minLength: 2, maxLength: 50 }
```

## Configuration

### **Global Configuration**
```typescript
import { StreamlinedValidationEngine } from '@/utilities/validation';

const engine = StreamlinedValidationEngine.getInstance({
  enableXSSProtection: true,
  enableSQLInjectionProtection: true,
  enableCommandInjectionProtection: true,
  enableSanitization: true,
  trimWhitespace: true,
  strictMode: false,
  debug: false
});
```

### **Available Options**
- `enableXSSProtection`: Enable XSS detection (default: true)
- `enableSQLInjectionProtection`: Enable SQL injection detection (default: true)
- `enableCommandInjectionProtection`: Enable command injection detection (default: true)
- `enableSanitization`: Enable input sanitization (default: true)
- `trimWhitespace`: Auto-trim whitespace (default: true)
- `allowEmpty`: Allow empty values when not required (default: false)
- `strictMode`: Enable strict validation mode (default: false)
- `debug`: Enable debug logging (default: DEV mode)

## Validation Results

### **Result Format**
```typescript
interface ValidationResult {
  valid: boolean;           // Whether validation passed
  errors: string[];         // Array of error messages
  sanitizedValue?: any;     // Cleaned/sanitized value
  originalValue?: any;      // Original input value
}
```

### **Form Validation Results**
```typescript
interface FormValidationResult {
  isValid: boolean;                    // Overall form validity
  errors: Record<string, string[]>;    // Field-specific errors
  sanitizedData?: Record<string, any>; // Cleaned form data
}
```

## Best Practices

### **1. Use Type-Specific Validation**
```typescript
//  Good: Use specific types
{ type: 'email', required: true }
{ type: 'password', required: true }

//  Avoid: Generic validation for specific formats
{ type: 'string', pattern: /email-regex/ }
```

### **2. Leverage Tree-Shaking**
```typescript
//  Good: Import specific validators for small bundles
import { FormatValidators } from '@/utilities/validation/validators/format-validators';

//  Okay: Convenience imports (slightly larger bundle)
import { validateEmail } from '@/utilities/validation';
```

### **3. Handle Validation Results Properly**
```typescript
//  Good: Check validity and handle errors
const result = validate(value, rules);
if (!result.valid) {
  setErrors(result.errors);
  return;
}
// Use result.sanitizedValue for further processing
```

### **4. Use Form-Level Validation**
```typescript
//  Good: Validate entire forms at once
const formResult = validateForm(formData, schema);
if (formResult.isValid) {
  // Submit formResult.sanitizedData
}
```
