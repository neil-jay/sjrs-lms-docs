## Overview

This guide covers all testing aspects of the SJRS LMS, including unit testing, integration testing, session management testing, and security feature validation.

## Testing Components

### Session Management Testing
The SJRS LMS includes comprehensive session management testing components for validating cross-tab synchronization and security features.

**Location**: `src/testing/session-test.tsx`

**Features:**
- Test mode toggle (1-minute timeout for faster testing)
- Real-time session status display
- Tab visibility and focus monitoring
- Activity simulation
- Testing instructions

**Usage:**
```tsx
import { SessionTest } from '../testing/session-test';

// Add to any page for testing
<SessionTest />
```

### Security Features Demo
Console-based demonstrations of security features for validation and debugging.

**Location**: `src/testing/security-demo.ts`

**Features:**
- Rate limiting demonstrations
- Error handling examples
- Security event logging
- Comprehensive security overview

**Usage:**
```tsx
import { demonstrateSecurityFeatures } from '../testing/security-demo';

// Run in browser console or component
demonstrateSecurityFeatures();
```

### Testing Module Exports
Centralized exports for all testing components and utilities.

**Location**: `src/testing/index.ts`

## Testing Scenarios

### Session Management Tests
- **Normal Operation**: 15-minute timeout, 2-minute warning
- **Test Mode**: 1-minute timeout, 30-second warning
- **Tab Switching**: Verify session continues in background
- **Focus Changes**: Test window focus/blur detection
- **Activity Simulation**: Reset timer with button clicks

### Security Feature Tests
- **Rate Limiting**: Test failed login attempts
- **Error Handling**: Verify user-friendly error messages
- **Event Logging**: Monitor security events in console
- **Session Cleanup**: Ensure proper logout and cleanup

### API Response Tests
- **Response Format**: Verify standardized response structure
- **Error Codes**: Test proper error code usage
- **CORS Headers**: Validate CORS implementation
- **Request Tracking**: Verify request ID generation

## Manual Testing Procedures

### 1. Authentication Flow
1. **Registration**: Test new user registration
2. **Login**: Verify login with valid credentials
3. **Invalid Login**: Test failed login attempts
4. **Session Timeout**: Wait for session expiry
5. **Logout**: Test proper logout cleanup

### 2. Session Management
1. **Multiple Tabs**: Open multiple tabs, test sync
2. **Focus Changes**: Switch between apps
3. **Activity Detection**: Test user activity tracking
4. **Timeout Warning**: Verify warning modal
5. **Extension**: Test session extension

### 3. API Functionality
1. **CRUD Operations**: Test Create, Read, Update, Delete
2. **Pagination**: Test list endpoints with pagination
3. **Filtering**: Test search and filter functionality
4. **Error Handling**: Test error scenarios
5. **Permission Checks**: Test role-based access

## Debugging Tests

### Session Management Debug
```tsx
// Enable debug mode
window.SESSION_DEBUG = true;

// Monitor session events
window.addEventListener('session-event', (event) => {
  console.log('Session Event:', event.detail);
});
```

### API Debug
```tsx
// Monitor API responses
const originalFetch = window.fetch;
window.fetch = async (...args) => {
  const response = await originalFetch(...args);
  const clone = response.clone();
  const data = await clone.json();
  console.log('API Response:', data);
  return response;
};
```
