## Refactoring Criteria

A module is considered **refactored** when it has:
1. Feature components in `src/components/features/[feature]/`
2. Pages organized in `src/pages/[feature]/` with proper structure (components/, hooks/, types/, utils/)
3. Backend API organized in `functions/api/[feature]/` with base/, handlers/, index.ts pattern

## Refactoring Checklist Template

For each module, complete:

- [ ] Create `src/components/features/[feature]/` directory
- [ ] Extract reusable components from pages
- [ ] Create `components/` subdirectory in feature folder
- [ ] Create `constants/` subdirectory (if needed)
- [ ] Create `hooks/` subdirectory (if needed)
- [ ] Create `types/` subdirectory (if needed)
- [ ] Create `utils/` subdirectory (if needed)
- [ ] Create `index.ts` barrel export
- [ ] Create `index.md` documentation
- [ ] Update page imports to use feature components
- [ ] Verify backend API structure matches pattern
- [ ] Test all functionality
- [ ] Update documentation
