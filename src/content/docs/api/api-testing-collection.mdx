This document provides comprehensive testing resources for the SJRS LMS API, including Postman collection, cURL examples, and testing procedures.

## 📦 Postman Collection

### **Import Instructions**
1. Download the [SJRS LMS API Collection](./sjrs-lms-api.postman_collection.json)
2. Import into Postman
3. Set up environment variables (see Environment Setup below)

### **Environment Variables**
```json
{
  "base_url": "https://sjrslms.jeevs.workers.dev",
  "origin": "https://sjrslms.jeevs.workers.dev",
  "email": "test@example.com",
  "password": "testpassword123",
  "user_id": ""
}
```

## 🖥️ cURL Examples

### **One-Time Setup**
```bash
curl -i -c cookies.txt -X POST "https://sjrslms.jeevs.workers.dev/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "password123"
  }'

curl -i -b cookies.txt -c cookies.txt "https://sjrslms.jeevs.workers.dev/api/auth/csrf"

xsrf=$(grep -E "\tXSRF-TOKEN\t" cookies.txt | tail -n 1 | awk '{print $7}')
```

## 🧪 Testing Procedures

### **1. Authentication Testing**

#### **Test User Registration**
```bash
# 1. Register new user
curl -X POST "https://sjrslms.jeevs.workers.dev/api/auth/register" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "testuser@example.com",
    "password": "testpass123",
    "first_name": "Test",
    "last_name": "User",
    "user_type": "student"
  }'

# Expected: 201 Created with user data
```

#### **Test User Login**
```bash
# 2. Login with registered user
curl -i -c cookies.txt -X POST "https://sjrslms.jeevs.workers.dev/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "testuser@example.com",
    "password": "testpass123"
  }'

# Expected: 200 OK with user data (auth cookie is set)
```

#### **Test Protected Endpoint**
```bash
# 3. Bootstrap CSRF (required for unsafe methods)
curl -i -b cookies.txt -c cookies.txt "https://sjrslms.jeevs.workers.dev/api/auth/csrf"

# 4. Test protected endpoint
curl -b cookies.txt "https://sjrslms.jeevs.workers.dev/api/auth/me"

# Expected: 200 OK with user profile data
```

   - Name: `SJRS LMS API - Development`
   - Base URL: `https://sjrslms.jeevs.workers.dev`

2. **Set Variables**
   ```
   base_url: https://sjrslms.jeevs.workers.dev
   origin: https://sjrslms.jeevs.workers.dev
   email: test@example.com
   password: testpass123
   user_id: (set after login, optional)
   ```

3. **Pre-request Scripts**
   ```javascript
   // Use the bundled collection pre-request script:
   // - removes Authorization header
   // - sets Origin header
   // - sets X-XSRF-TOKEN header from XSRF-TOKEN cookie (if present)
   ```

### **Automated Testing with Newman**

```bash
# Install Newman
npm install -g newman

# Run collection
newman run sjrs-lms-api.postman_collection.json \
  -e sjrs-lms-dev.postman_environment.json \
  --reporters cli,json \
  --reporter-json-export results.json
```

### **Continuous Integration Testing**

```yaml
# GitHub Actions example
name: API Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Install Newman
        run: npm install -g newman
      - name: Run API Tests
        run: |
          newman run docs/api/sjrs-lms-api.postman_collection.json \
            -e docs/api/sjrs-lms-dev.postman_environment.json \
            --reporters cli,json \
            --reporter-json-export test-results.json
      - name: Upload Test Results
        uses: actions/upload-artifact@v2
        with:
          name: api-test-results
          path: test-results.json
```

### **Common Issues & Solutions**

#### **CORS Errors**
```bash
# Check if CORS headers are present
curl -H "Origin: https://example.com" \
  -X GET "https://sjrslms.jeevs.workers.dev/api/health" \
  -v
```

#### **Token Expiration**
```bash
# Check session validity
curl -v -b cookies.txt "https://sjrslms.jeevs.workers.dev/api/auth/me"
```

#### **Rate Limiting**
```bash
# Test rate limiting
for i in {1..10}; do
  curl -b cookies.txt "https://sjrslms.jeevs.workers.dev/api/books"
```
