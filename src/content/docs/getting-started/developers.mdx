# 💻 Getting Started for Developers

Welcome to the technical documentation for the SJRS Library Management System. This guide covers the architecture, setup, and development workflows.

## 🏗️ Architecture Overview

The system is built on a **Serverless Architecture** using the Cloudflare stack:

*   **Frontend**: React (Vite) + TypeScript + Ant Design
*   **Backend**: Cloudflare Workers (Edge Compute)
*   **Database**: Cloudflare D1 (SQLite at the Edge)
*   **Storage**: Cloudflare R2 (Object Storage)
*   **Auth**: Custom JWT-based authentication with HttpOnly cookies

## 🛠️ Development Setup

### **Prerequisites**
*   Node.js (Latest LTS)
*   npm or pnpm
*   Wrangler CLI (`npm install -g wrangler`)

### **1. Clone & Install**
```bash
git clone https://github.com/your-org/sjrs-lms.git
cd sjrs-lms
npm install
```

### **2. Environment Configuration**
Copy the example environment file:
```bash
cp .env.example .env
```
Update the variables with your local or staging credentials.

### **3. Database Setup (Local)**
Initialize the local D1 database:
```bash
npm run db:migrate:local
npm run db:seed:local
```

### **4. Run Development Server**
```bash
npm run dev
```
The frontend will be available at `http://localhost:5173` (or similar).

## 📂 Project Structure

*   `/src`: Frontend React application
*   `/functions`: Backend Cloudflare Workers API
*   `/sql`: Database schema and migrations
*   `/shared`: Shared types and utilities

## 🧪 Testing

*   **Unit Tests**: `npm run test`
*   **Integration Tests**: `npm run test:integration`
