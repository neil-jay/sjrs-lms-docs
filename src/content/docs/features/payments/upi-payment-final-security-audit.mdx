4. Error handling (catches violations)

## Edge Cases - All Handled

### Race Conditions
- **Concurrent payment completion:** Atomic SQL prevents double completion
- **Transaction ID reuse:** Database unique index prevents duplicates
- **Concurrent updates:** Conditional WHERE clauses prevent conflicts

### Missing Data
- **User missing:** Returns 401 (authentication required)
- **Penalty owner missing:** Returns 404 (not found)
- **Penalty already paid:** Returns 404 (not found or already paid)
- **Payment not found:** Returns 404

### Type Coercion
- **User ID comparison:** Uses String() conversion
- **Amount comparison:** Uses Number() with tolerance
- **All comparisons:** Type-safe

### Authorization Edge Cases
- **User owns payment:** Can update
- **User doesn't own payment:** Needs payments:update permission
- **No authentication:** Returns 401
- **Invalid permissions:** Returns 403
