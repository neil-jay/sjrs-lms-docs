## Overview
This module has been refactored to improve modularity, align with the backend schema, and eliminate code duplication.

## File Structure
```
src/pages/borrow-limits/
├── index.tsx              # Main export file
├── BorrowLimitForm.tsx    # Shared form component
```

## Key Improvements

### 1. **Modular Architecture**
- **Separated concerns**: Each component is now in its own file
- **Shared components**: `BorrowLimitForm` eliminates duplication between create/edit
- **Clean exports**: Main index file provides clean API

### 2. **Backend Alignment**
- **Field mapping**: Updated to use `loan_period_days` instead of `loan_duration_days`
- **Schema consistency**: Frontend now matches backend database schema
- **Type safety**: Proper TypeScript interfaces aligned with backend

### 3. **Code Quality**
- **Eliminated duplication**: Shared form component reduces maintenance
- **Consistent validation**: All forms use the same validation rules
- **Better error handling**: Consistent error handling across components

### 4. **User Experience**
- **Year-based limits**: Support for Student_1, Student_2, Student_3, Student_4
- **Proper validation**: Form validation with meaningful error messages
- **Currency formatting**: Indian Rupee (₹) formatting for fine amounts

## Database Schema Changes

### Migration Required
The backend requires a migration to update the `borrow_limits` table:
- `sql/migrations/update-borrow-limits-schema.sql`

### New Schema
```sql
CREATE TABLE borrow_limits (
  user_type VARCHAR(20) PRIMARY KEY
    CHECK (user_type IN ('Student_1','Student_2','Student_3','Student_4','Professor','Guest')),
  max_books INTEGER NOT NULL CHECK (max_books > 0),
  loan_period_days INTEGER NOT NULL CHECK (loan_period_days > 0),
  max_renewals INTEGER NOT NULL CHECK (max_renewals >= 0),
  fine_per_day DECIMAL(10,2) NOT NULL CHECK (fine_per_day >= 0),
  is_active BOOLEAN DEFAULT TRUE,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## Validation Rules
All forms use consistent validation from `src/constants/borrow-limits.ts`:
- **Max Books**: 1-50
- **Loan Period**: 1-365 days
- **Max Renewals**: 0-365
- **Fine per Day**: ₹0-100

## User Types Supported
- **Student_1**: 1st year students (5 books, 14 days, 2 renewals)
- **Student_2**: 2nd year students (10 books, 14 days, 2 renewals)
- **Student_3**: 3rd year students (12 books, 14 days, 2 renewals)
- **Student_4**: 4th year students (15 books, 14 days, 2 renewals)
- **Professor**: Faculty (10 books, 30 days, 3 renewals)
- **Guest**: Visitors (2 books, 7 days, 1 renewal)

## Hardcoded Fallback Values

The system uses hardcoded fallback values when the database has no configured limits. These are defined in `src/constants/borrow-limits.ts`:

### Default Fallback
Used when no specific user type configuration exists:
- **Max Books**: 5
- **Loan Period**: 14 days
- **Max Renewals**: 2
- **Fine per Day**: ₹1.00

### Student Limits (Year-Based)

| Year | Max Books | Loan Period | Renewals | Fine/Day |
|------|-----------|-------------|----------|----------|
| **Student_1** (1st year) | 5 | 14 days | 2 | ₹25.00 |
| **Student_2** (2nd year) | 10 | 14 days | 2 | ₹25.00 |
| **Student_3** (3rd year) | 12 | 14 days | 2 | ₹25.00 |
| **Student_4** (4th year) | 15 | 14 days | 2 | ₹25.00 |

### Other User Types

| User Type | Max Books | Loan Period | Renewals | Fine/Day |
|-----------|-----------|-------------|----------|----------|
| **Professor** | 10 | 30 days | 3 | ₹50.00 |
| **Guest** | 2 | 7 days | 1 | ₹100.00 |

### Fallback Behavior

1. **Database First**: System attempts to fetch limits from `borrow_limits` table
2. **Fallback on Error**: If database query fails or returns no results, uses hardcoded values
3. **Silent Fallback**: No error shown to user when falling back to hardcoded values
4. **Override Anytime**: Database values always take precedence when configured

**Important**: To ensure consistent behavior across the system, configure limits in the database via the superuser dashboard at `/dashboard-superuser/borrow-limits`.
