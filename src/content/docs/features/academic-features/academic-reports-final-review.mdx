## Security Review

### SQL Injection Protection
- **Status**:  **SECURE**
- **Implementation**: All queries use parameterized statements with `.bind()`
- **Evidence**: 
  ```typescript
  WHERE l.borrowed_at >= ? AND l.borrowed_at < ?
  ${userId ? 'AND l.user_id = ?' : ''}
  `).bind(startDate, endDate, ...(userId ? [userId] : []));
  ```
- **No vulnerabilities found**

### Authorization & Permissions
- **Status**:  **SECURE**
- **Backend**: Permission check in `functions/api/reports/index.ts` (requires `reports:read`)
- **Frontend**: Permission check before rendering page
- **No bypass vulnerabilities found**

### Input Validation
- **Status**:  **SECURE**
- **Academic Year**: Required, validated, falls back to global
