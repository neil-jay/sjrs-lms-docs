## Components Overview

### Core Components (`/components`)

#### 1. **UserList** - Main user listing component
- **Purpose**: Display users in a table with filtering, searching, and actions
- **Features**: 
  - Search by name, email
  - Filter by user type, status, role
  - Sortable columns
  - Bulk operations (for superusers)
  - Action menu for each user
- **Usage**:
```tsx
import { UserList } from './components/features/users';

<UserList
  onView={(user) => console.log('View user', user)}
  onEdit={(user) => console.log('Edit user', user)}
  onDelete={(user) => console.log('Delete user', user)}
  onCreate={() => console.log('Create new user')}
  showActions={true}
  showCreateButton={true}
/>
```

#### 2. **UserForm** - User creation and editing form
- **Purpose**: Form component for creating/editing users
- **Features**:
  - User type selection (Student, Professor, Guest)
  - Role assignment (with permission checks)
  - Student-specific fields (registration number, year of study)
  - Form validation
  - Status management
- **Usage**:
```tsx
import { UserForm } from './components/features/users';

// For editing
<UserForm
  user={existingUser}
  mode="edit"
  onSuccess={() => console.log('User updated')}
  onCancel={() => console.log('Cancelled')}
/>

// For creating
<UserForm
  mode="create"
  onSuccess={() => console.log('User created')}
  onCancel={() => console.log('Cancelled')}
/>
```

#### 3. **UserView** - User details display
- **Purpose**: Read-only display of user information
- **Features**:
  - User profile information
  - Status indicators
  - Role badges
  - Student-specific details
  - Account status overview
- **Usage**:
```tsx
import { UserView } from './components/features/users';

<UserView
  user={userData}
  visible={true}
  onClose={() => setVisible(false)}
  showModal={true} // or false for inline display
/>
```

#### 4. **UserCreate** - User creation wrapper
- **Purpose**: Modal wrapper for UserForm in create mode
- **Usage**:
```tsx
import { UserCreate } from './components/features/users';

<UserCreate
  visible={showCreateModal}
  onClose={() => setShowCreateModal(false)}
  onSuccess={() => {
    setShowCreateModal(false);
    refetchUsers();
  }}
/>
```

#### 5. **UserEdit** - User editing wrapper
- **Purpose**: Modal wrapper for UserForm in edit mode
- **Usage**:
```tsx
import { UserEdit } from './components/features/users';

<UserEdit
  user={selectedUser}
  visible={showEditModal}
  onClose={() => setShowEditModal(false)}
  onSuccess={() => {
    setShowEditModal(false);
    refetchUsers();
  }}
/>
```

#### 6. **UserDetailsDrawer** - Comprehensive user details sidebar
- **Purpose**: Drawer component showing detailed user information and actions
- **Features**:
  - User profile with avatar
  - Library activity statistics
  - Quick actions (edit, delete, view activity)
  - Student information display
  - Account status indicators
- **Usage**:
```tsx
import { UserDetailsDrawer } from './components/features/users';

<UserDetailsDrawer
  user={selectedUser}
  visible={showDrawer}
  onClose={() => setShowDrawer(false)}
  onEdit={(user) => handleEdit(user)}
  onDelete={(user) => handleDelete(user)}
  onViewActivity={(user) => handleViewActivity(user)}
/>
```

## Integration with Existing System

### API Integration
These components are designed to work with:
- **D1 Database**: Using existing `useD1UsersQuery`, `useD1UpdateUser` hooks
- **User Service**: `src/services/user.service.ts`
- **Authentication**: `useD1Auth` context
- **Permissions**: `useRoles` and `canManageRoles` utilities

### Page Integration
The components can be used in:
- `/pages/members/` - Replace existing member management
- `/pages/students/` - Student-specific user management
- `/pages/professors/` - Professor-specific user management
- `/pages/guests/` - Guest user management

### Example Integration in a Page
```tsx
import React, { useState } from 'react';
import { UserList, UserCreate, UserEdit, UserDetailsDrawer, UserActivityModal } from '../components/features/users';

export const UserManagementPage: React.FC = () => {
  const [selectedUser, setSelectedUser] = useState(null);
  const [showCreateModal, setShowCreateModal] = useState(false);
  const [showEditModal, setShowEditModal] = useState(false);
  const [showDrawer, setShowDrawer] = useState(false);
  const [showActivityModal, setShowActivityModal] = useState(false);

  return (
    <div>
      <UserList
        onView={(user) => {
          setSelectedUser(user);
          setShowDrawer(true);
        }}
        onEdit={(user) => {
          setSelectedUser(user);
          setShowEditModal(true);
        }}
        onCreate={() => setShowCreateModal(true)}
      />
      
      <UserCreate
        visible={showCreateModal}
        onClose={() => setShowCreateModal(false)}
        onSuccess={() => setShowCreateModal(false)}
      />
      
      <UserEdit
        user={selectedUser}
        visible={showEditModal}
        onClose={() => setShowEditModal(false)}
        onSuccess={() => setShowEditModal(false)}
      />
      
      <UserDetailsDrawer
        user={selectedUser}
        visible={showDrawer}
        onClose={() => setShowDrawer(false)}
        onViewActivity={(user) => {
          setSelectedUser(user);
          setShowActivityModal(true);
        }}
      />
      
      <UserActivityModal
        user={selectedUser}
        visible={showActivityModal}
        onClose={() => setShowActivityModal(false)}
      />
    </div>
  );
};

## Permission System Integration

The components respect the existing permission system:
- **Role Management**: Only superusers can assign roles
- **User Status**: Only users with appropriate permissions can change status
- **Delete Operations**: Restricted based on user roles
- **Self-editing**: Users can edit their own profiles with limited fields

## Data Flow

1. **UserList** displays all users with filtering capabilities
2. **User actions** trigger appropriate modal/drawer components
3. **Forms** use existing hooks for data mutations
4. **Success callbacks** trigger data refetching and UI updates
5. **Error handling** uses the unified error system

## Styling & Theming

All components use:
- Ant Design components for consistency
- Existing color schemes and status mappings
- Responsive design patterns
- Dark/light mode support through existing theme context
