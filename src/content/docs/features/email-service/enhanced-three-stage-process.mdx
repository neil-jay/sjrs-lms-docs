## Overview

The SJRS LMS now features an enhanced three-stage user registration process that ensures complete user information, better security, and improved user experience. This process replaces the previous two-stage system with a more comprehensive approach.

## 🎯 Process Flow

### Stage 1: Email Confirmation
- **Action**: User registers with email and password
- **System**: Sends confirmation email with three-stage process explanation
- **User**: Clicks confirmation link to verify email
- **Status**: Account status remains `pending`; onboarding status is `pending_email_confirmation` until email verification

### Stage 2: Profile Completion
- **Action**: User completes profile with required information
- **Requirements**:
  - **Students**: Registration number, year of study, phone
  - **Professors**: Phone number, stream
  - **Guests**: Phone number, institution/organization
- **System**: Sends profile completion notification email
- **Status**: Account status remains `pending`; onboarding status changes from `profile_incomplete` to `pending_approval` after profile completion

### Stage 3: Admin Approval
- **Action**: Admin reviews user profile and information
- **Options**: Approve or reject the account
- **System**: Sends approval/rejection notification email
- **Status**: Account status changes to `active` (approved) or `inactive` (rejected). Onboarding status becomes `complete`.

## 🚀 Usage Examples

### Email Confirmation Flow
```typescript
// User registers
const user = await createUser({
  email: 'user@example.com',
  password: 'securepassword',
  first_name: 'John',
  last_name: 'Doe',
  user_type: 'Student',
  status: 'pending'
}, env);

// System sends confirmation email
await sendConfirmationEmail(user.email, user.first_name, token, env);
```

### Profile Completion Flow
```typescript
// User completes profile
const profileData = {
  user_id: 123,
  user_type: 'Student',
  phone: '+1-555-0123',
  stream: 'Computer Science',
  registration_number: '2024',
  year_of_study: 2
};

// System sends profile completion email
await sendProfileCompletionEmail(
  user.email,
  user.first_name,
  profileData.user_type,
  profileData,
  env
);
```

### Admin Approval Flow
```typescript
// Admin approves user
await sendAdminApprovalEmail(
  user.email,
  user.first_name,
  admin.name,
  'approved',
  undefined,
  env
);

// Admin rejects user
await sendAdminApprovalEmail(
  user.email,
  user.first_name,
  admin.name,
  'rejected',
  'Incomplete profile information',
  env
);
```
