## Overview

Fixed the confusion where **loan and payment transactions were incorrectly creating System Notifications** when they should be creating **App Notifications** (user-specific notifications).

---

## Changes Made

### 1. Created App Notifications Helper (`functions/lib/notifications/app-notifications.ts`)

**New File**: Centralized utility for creating user-specific App Notifications

**Functions:**
- `createAppNotification()` - Creates a single App Notification in the `notifications` table
- `createBulkAppNotifications()` - Creates multiple App Notifications in bulk

**Features:**
- Type-safe with TypeScript
- Proper error handling
- Supports all notification fields (type, priority, actionUrl, metadata)
- Inserts into `notifications` table (not `notification_events`)

### 2. Updated Transaction Notifications (`functions/lib/notifications/transaction-notifications.ts`)

**Changes:**
- **Removed**: System Notification creation (`createNotificationEvent`)
- **Added**: App Notification creation (`createAppNotification`)
- **Updated**: All three functions now create App Notifications:
  - `publishLoanCreatedNotification()` - Creates App Notification when loan is created
  - `publishLoanStatusNotification()` - Creates App Notification when loan status changes
  - `publishPaymentReceiptNotification()` - Creates App Notification when payment is received

**Key Improvements:**
- Notifications now go to `notifications` table (user-specific)
- Proper notification types: `success`, `warning`, `error`, `info`, `reminder`
- Proper priority levels: `low`, `normal`, `high`, `urgent`
- Action URLs point to user dashboard pages (`/dashboard/loans`, `/dashboard/payments`)
- Metadata preserved for transaction tracking

### 3. Notification Type Mapping

**Loan Status → Notification Type:**
- `returned` → `success` (normal priority)
- `overdue` (≥14 days) → `error` (urgent priority)
- `overdue` (&lt;14 days) → `warning` (high priority)
- `approved` → `success` (normal priority)
- `rejected` → `warning` (normal priority)
- Other statuses → `info` (normal priority)

**Payment Receipt:**
- Always → `success` (normal priority)

**Loan Created:**
- Always → `success` (normal priority)

## Impact

### What's Fixed

1. **Correct Notification Type**: Loans and payments now create App Notifications (user-specific)
2. **Proper Storage**: Notifications stored in `notifications` table, not `notification_events`
3. **User-Specific**: Each notification is tied to a specific user (`user_id`)
4. **Better UX**: Action URLs point to relevant dashboard pages
5. **Type Safety**: Proper TypeScript types for notification types and priorities

### What Remains Unchanged

1. **Email Notifications**: Still sent as before (no changes)
2. **Handler Functions**: All handlers (`create-loan.ts`, `update-loan.ts`, `create-receipt.ts`) work the same
3. **Function Signatures**: No breaking changes to public APIs
4. **Error Handling**: Same error handling patterns

---

## Database Impact

### Before (Incorrect)
- Loan/payment transactions → `notification_events` table (System Notifications)
- One event record with user targeting
- Not user-specific

### After (Correct)
- Loan/payment transactions → `notifications` table (App Notifications)
- One notification record per user
- User-specific, tied to `user_id`
