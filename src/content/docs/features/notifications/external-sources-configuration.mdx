## 1. Deployment Pipeline (Release Pipeline)

### ✅ Code Implementation
- **Location**: `scripts/auto-version.js`
- **Function**: `notifyDeploymentEvent()`
- **Called**: After successful/failed deployments (lines 496-504, 518-524)
- **Status**: ✅ **Fully implemented**

### ⚠️ Configuration Required

#### Environment Variables Needed:
```bash
# Option 1: Global token (works for all sources)
NOTIFICATION_INGEST_TOKEN=your_secure_token_here

# Option 2: Source-specific token (recommended)
NOTIFICATION_INGEST_TOKEN_RELEASE=your_release_pipeline_token_here
```

#### Endpoint Configuration (Optional):
```bash
# Default: https://your-api-domain.com/api/notifications/events/ingest
# Override if needed:
NOTIFICATION_INGEST_ENDPOINT=https://your-api-domain.com/api/notifications/events/ingest
```

#### Current Behavior:
- ✅ Code is integrated into deployment script
- ✅ Automatically called after deployments
- ⚠️ **If token is missing**: Logs message and skips notification (non-blocking)
- ⚠️ **If token is present**: Sends notification to ingest endpoint

#### How to Configure:
1. Set `NOTIFICATION_INGEST_TOKEN` or `NOTIFICATION_INGEST_TOKEN_RELEASE` in your deployment environment
2. Token should be stored as a secret in Cloudflare Workers (for production)
3. For GitHub Actions, add to repository secrets
4. For local testing, add to `.env` file

#### Verification:
- Check if token is set: `echo $NOTIFICATION_INGEST_TOKEN`
- Check deployment logs for: `📣 Posted deployment update to system notifications feed.`
- If you see: `ℹ️ Notification ingest token is not configured; skipping system notification.` → Token is missing

## 2. Sentry

### ✅ Code Implementation
- **Location**: `functions/api/notifications/handlers/ingest-notification-event.ts`
- **Handler**: `buildSentryEvent()` (lines 286-365)
- **Source**: `monitoring:sentry`
- **Status**: ✅ **Code ready, webhook not configured**

### ❌ Configuration Required

#### Environment Variables Needed:
```bash
# Option 1: Global token
NOTIFICATION_INGEST_TOKEN=your_secure_token_here

# Option 2: Source-specific token (recommended)
NOTIFICATION_INGEST_TOKEN_SENTRY=your_sentry_webhook_token_here
```

#### Sentry Webhook Configuration:
1. **Go to Sentry Dashboard** → Project Settings → Webhooks
2. **Add Webhook URL**: `https://your-api-domain.com/api/notifications/events/ingest`
3. **Add Header**: `x-notification-ingest-token: your_sentry_webhook_token_here`
4. **Select Events**:
   - Issue created
   - Issue resolved
   - Issue updated
   - Event created (for critical errors)

#### Webhook Payload Format:
Sentry will send events in their format. The ingest handler automatically converts:
- Sentry level → Notification severity
- Sentry event → Notification event
- Sentry project → Notification metadata

#### How to Configure:
1. **Set token in Cloudflare Workers secrets**:
   ```bash
   wrangler secret put NOTIFICATION_INGEST_TOKEN_SENTRY
   ```

2. **Configure Sentry Webhook**:
   - URL: `https://your-api-domain.com/api/notifications/events/ingest`
   - Header: `x-notification-ingest-token: <your-token>`
   - Events: Select relevant events (errors, issues)

3. **Test**: Trigger a test error in Sentry and verify notification appears

#### Verification:
- Check Sentry webhook logs in Sentry dashboard
- Check System Notifications in `/dashboard-superuser/system-notifications`
- Look for notifications with source: `monitoring:sentry`

## 3. Cloudflare Analytics

### ✅ Code Implementation
- **Location**: `functions/api/notifications/handlers/ingest-notification-event.ts`
- **Handler**: `buildAnalyticsEvent()` (lines 367-447)
- **Source**: `analytics:cloudflare`
- **Status**: ✅ **Code ready, webhook not configured**

### ❌ Configuration Required

#### Environment Variables Needed:
```bash
# Option 1: Global token
NOTIFICATION_INGEST_TOKEN=your_secure_token_here

# Option 2: Source-specific token (recommended)
NOTIFICATION_INGEST_TOKEN_CLOUDFLARE=your_cloudflare_analytics_token_here
```

#### Cloudflare Analytics Webhook Configuration:
**Note**: Cloudflare Analytics doesn't have native webhooks. You'll need to:

1. **Use Cloudflare Workers Cron Triggers** (recommended):
   - Create a scheduled worker that checks analytics metrics
   - If thresholds are breached, call the ingest endpoint
   - Run every 5-15 minutes

2. **Use External Monitoring Service**:
   - Set up external service (e.g., UptimeRobot, Pingdom) to monitor metrics
   - Configure webhook to call ingest endpoint when thresholds are breached

3. **Use Cloudflare GraphQL Analytics API**:
   - Create a worker that queries analytics API periodically
   - Compare metrics against thresholds
   - Call ingest endpoint when thresholds are breached

#### Example Worker for Analytics Monitoring:
```typescript
// functions/analytics-monitor.ts
export default {
  async scheduled(event: ScheduledEvent, env: Environment) {
    // Query Cloudflare Analytics API
    const metrics = await fetchAnalyticsMetrics(env);
    
    // Check thresholds
    if (metrics.errorRate > 0.05) {
      await fetch(`${env.API_BASE_URL}/api/notifications/events/ingest`, {
        method: 'POST',
        headers: {
          'x-notification-ingest-token': env.NOTIFICATION_INGEST_TOKEN_CLOUDFLARE,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          source: 'analytics:cloudflare',
          title: 'High Error Rate Detected',
          message: `Error rate is ${metrics.errorRate}% (threshold: 5%)`,
          category: 'incident',
          severity: 'warning',
          analytics: {
            errorRate: metrics.errorRate,
            requestCount: metrics.requestCount,
            timestamp: new Date().toISOString(),
          },
        }),
      });
    }
  },
};
```

#### How to Configure:
1. **Set token in Cloudflare Workers secrets**:
   ```bash
   wrangler secret put NOTIFICATION_INGEST_TOKEN_CLOUDFLARE
   ```

2. **Set up monitoring** (choose one):
   - Option A: Create scheduled worker (recommended)
   - Option B: Use external monitoring service
   - Option C: Use Cloudflare GraphQL Analytics API

3. **Test**: Trigger a threshold breach and verify notification appears

#### Verification:
- Check System Notifications for source: `analytics:cloudflare`
- Verify notifications appear when thresholds are breached

## Security Notes

1. **Token Security**:
   - Never commit tokens to git
   - Use Cloudflare Workers secrets for production
   - Use different tokens for each source (recommended)
   - Rotate tokens periodically

2. **Endpoint Security**:
   - Ingest endpoint requires token authentication
   - Tokens are validated before processing
   - Failed authentication returns 401 Unauthorized

3. **Rate Limiting**:
   - Consider adding rate limiting to ingest endpoint
   - Monitor for abuse
   - Set up alerts for unusual activity

## Troubleshooting

### Deployment Notifications Not Appearing
1. Check if token is set: `echo $NOTIFICATION_INGEST_TOKEN`
2. Check deployment logs for notification messages
3. Verify endpoint URL is correct
4. Check network connectivity during deployment
5. Verify token is valid (test with curl)

### Sentry Notifications Not Appearing
1. Check Sentry webhook logs
2. Verify webhook URL is correct
3. Verify token header is set correctly
4. Check ingest endpoint logs for errors
5. Test webhook manually with curl

### Cloudflare Analytics Notifications Not Appearing
1. Verify monitoring worker is running
2. Check worker logs for errors
3. Verify token is set correctly
4. Check if thresholds are being breached
5. Test ingest endpoint manually
